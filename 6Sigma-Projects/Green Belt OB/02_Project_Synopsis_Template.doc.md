# PROJECT SYNOPSIS - PROJECT NLAP
**Project Title:** Reduction in Outbound Percentage – Project NLAP (Workgroup-level)
**Project Lead:** **Rahul Nethikar** (Ops Team Leader)
**Project Sponsor:** **Operations Manager / Workgroup Manager**
**Date:** April 2009

---

## 2. BUSINESS CASE
Verizon DSL Technical Support (Project NLAP) is seeing a high volume of outbound calls that are eating into our inbound capacity. While many of these are essential—**specifically the Merchantile Line Test (MLT) callbacks which are a valid and necessary part of our technical resolution process**—we are still seeing a high percentage of "unnecessary" dialing. This includes agents finishing calls early by promising a callback for non-technical reasons or using outbound to park complex issues. We need to preserve the valid MLT process while cutting out the waste to hit our 15% target.

## 3. PROBLEM STATEMENT
Currently, Project NLAP is running at a **21% outbound rate**. A deep dive shows that while MLT-related callbacks are a fixed requirement, there is a significant volume of non-essential outbound calls being made. This variance from our 15% target is causing a 6% capacity loss, which translates to higher headcount costs and delayed response times for waiting customers.

## 4. GOAL STATEMENT
Reduce the overall outbound call percentage for Project NLAP from **21%** to **15% or less** by the end of Q3 2009. We will maintain all **valid MLT callback protocols** while eliminating non-process driven outbound volume.

## 5. PROJECT SCOPE
*   **In-Scope:** This project was executed specifically within a single workgroup (Verizon DSL Floor) and was not implemented across the entire contact center. 
*   **Workgroup-level implementation:** Process changes, governance (PIN), and monitoring were restricted exclusively to the target workgroup floor.
